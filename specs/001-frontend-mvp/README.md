## Notes

**Architecture Decision**: Hexagonal pattern chosen to match backend Clean Architecture and enable testability. Frontend uses React Context + Zustand for dependency injection (functional equivalent of Rust ServiceContainer pattern).

**MVP Scope**: P3 (indexing) explicitly noted as convenience feature. Users can rely on CLI (`mdx index`) for initial indexing while benefiting from search interface (P1) immediately. This allows faster MVP delivery with complete value.

**Testing Approach**: Constitution requires >80% coverage. Testing strategy documented in plan.md Phase 1. Framework setup (Vitest, React Testing Library, Playwright) deferred to implementation phase.
